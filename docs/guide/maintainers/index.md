# :material-shield-account: For Maintainers

As a maintainer of a Power Platform open-source project, you play a critical
role in the ecosystem. Your work enables thousands of developers, citizen
makers, and organisations to build better solutions. This section provides
actionable guidance to help you run your project effectively, attract
contributors, and follow practices that make your repository welcoming and
sustainable.

---

## What Makes a Great Open-Source Maintainer?

!!! success "The fundamentals"
    Great maintainers share a few key traits:

    - **Responsiveness** — Acknowledge issues and pull requests quickly, even
      if you cannot act on them immediately.
    - **Clarity** — Write clear READMEs, label issues well, and keep
      contribution guidelines current.
    - **Inclusiveness** — Welcome newcomers with good-first-issue labels,
      encouraging language, and a visible Code of Conduct.
    - **Sustainability** — Set expectations about your availability and
      encourage co-maintainers so the project does not depend on one person.

## Quick Wins for Today

If you only have 15 minutes, these three actions will make the biggest
difference for your project's visibility and contributor experience:

1. **Add a tracked topic** to your repository so the Hub discovers it
   automatically — see [Repository Setup](repository-setup.md).
2. **Label one issue as `good first issue`** to invite newcomers — see
   [Community Health](community-health.md#labelling-issues).
3. **Add a `CONTRIBUTING.md` file** so contributors know how to help — see
   [Community Health](community-health.md#the-contributing-file).
