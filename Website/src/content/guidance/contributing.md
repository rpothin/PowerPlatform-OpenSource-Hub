## How to Contribute

Contributing to open-source projects is one of the best ways to learn, grow, and give back to the community. Here's how to get started with Power Platform projects.

## Before You Begin

1. **Read the Code of Conduct** — Every project has community guidelines. Respect them.
2. **Check Existing Issues** — Before opening a new issue, search to see if it already exists.
3. **Read the README** — Understand what the project does and how it's structured.

## Contribution Workflow

### 1. Fork the Repository

Click the "Fork" button on GitHub to create your own copy of the repository.

### 2. Clone Your Fork

```bash
git clone https://github.com/YOUR-USERNAME/REPO-NAME.git
cd REPO-NAME
```

### 3. Create a Branch

```bash
git checkout -b feature/your-feature-name
```

### 4. Make Your Changes

- Follow the project's coding standards.
- Write tests if applicable.
- Update documentation if needed.

### 5. Commit and Push

```bash
git add .
git commit -m "feat: describe your change"
git push origin feature/your-feature-name
```

### 6. Open a Pull Request

Go to the original repository and open a pull request from your fork. Provide a clear description of your changes.

## Best Practices

- **Keep PRs small and focused** — One change per pull request.
- **Write meaningful commit messages** — Use conventional commits format.
- **Test your changes** — Run the project's test suite before submitting.
- **Be responsive** — Address review feedback promptly.
- **Be patient** — Maintainers are often volunteers with limited time.

## Types of Contributions

Not all contributions require code! Consider:

- 📝 **Documentation** — Fix typos, improve guides, add examples.
- 🐛 **Bug Reports** — Report issues with clear reproduction steps.
- 💡 **Feature Requests** — Suggest improvements with detailed use cases.
- 🧪 **Testing** — Add or improve test coverage.
- 🌐 **Translations** — Help make projects accessible in other languages.

## Getting Help

If you're stuck or have questions:

- Open a discussion on the project's GitHub Discussions page.
- Join the [Power Platform Community](https://powerusers.microsoft.com/).
- Visit the [Maintainer Home](/maintainers/) for additional resources.
