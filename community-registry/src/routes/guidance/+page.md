# Guidance

Welcome to the Power Platform Open-Source Hub guidance section! This page provides resources and best practices for contributing to the ecosystem.

## Getting Started

The Power Platform ecosystem includes a wide range of open-source projects spanning Power Apps, Power Automate, Copilot Studio, Dataverse, and more.

### Finding the Right Project

1. **Browse the Registry** — Use the [Community Registry](/) to explore projects by language, topic, or focus area.
2. **Look for "Good First Issues"** — Many projects tag beginner-friendly issues. Look for the green badge on repository cards.
3. **Check "Help Wanted"** — These issues are actively seeking community contributions.

### Contributing Guidelines

Before contributing to any project, always:

- Read the project's **README** and **CONTRIBUTING** guide
- Check the project's **Code of Conduct**
- Look at existing **issues and pull requests** to avoid duplicating effort
- Start small with documentation fixes or bug reports

## Focus Areas

### Power Apps & PCF Controls

Power Apps projects include canvas apps, model-driven apps, and PCF (PowerApps Component Framework) controls. These projects often use **TypeScript** and **C#**.

### Power Automate & RPA

Power Automate projects cover cloud flows, desktop flows, and robotic process automation. You'll find connectors, templates, and automation tools.

### Copilot Studio

Copilot Studio (formerly Power Virtual Agents) projects focus on chatbots, AI-powered assistants, and conversational AI for the Power Platform.

### Dataverse

Dataverse projects involve data modeling, plugins, web resources, and tools for working with the Common Data Service.
